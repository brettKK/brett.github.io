加上volatile后，查看汇编指令，每次循环都会查询内存里的变量后。 因此产生了volatile(易变的)的编程语义， 所以每次读易变的变量时读内存，每次写易变的变量时写内存。

extern int volatile done;
void join() {
    while (!done) {
        ;
    }
}

int x = 0, y = 0;
void T1() {
    x = 1;
    asm volatile("" : : "memory");
    printf("y = %d\n", y); 
}
void T2() {
    y = 1;
    asm volatile("" : : "memory");
    printf("x = %d\n", x);
}
// pipe to head -n 1000000 |sort|uniq -c
// 可以看到 x = 0 y = 0 出现了...

next， 加上volatile后保证汇编层面上的语义正确。 但是在cpu内部维护指令依赖的DAG，不依赖的指令同时发射执行。 现在的处理器也是数据流分析器，编译器，做出应有的应对，它才可以跑得快。 所以printf y 可以在写入x 之前执行。可以在mfence保证指令执行顺序与代码编写顺序一致。

理解并发程序的执行 🔗

并发程序 = 多个执行流， 共享内存的状态机。

peterson算法。 A举旗子，往厕所门上贴“B”正在使用的标签。 B举旗子，往厕所门上贴“A”正在使用的标签。 当 对方的旗子举起来了，且门上的名字不是自己，则等待。其他情况，可以进。

int volatile x = 0, y = 0, turn = A;
void Ta() {
    while(1) {
        x = 1;
        turn = B;
        while( y && turn == B) ;
        critical_section();
        x = 0;
    }
}

void Tb() {
    while(1) {
        y = 1;
        turn = A;
        while( x && turn = A) ;
        critical_section();
        y = 0;
    }
}

画状态机（pc1, pc2, x, y, turn）。用工具画。 model checker。状态机可以抽象为图。可达性。

并发控制：互斥 🔗

实现互斥的根本困难是： 不能同时读/写内存。

如何在多处理器上实现线程互斥？

自旋锁(spin lock)。 🔗

硬件加指令。

xchg指令： 先看一下，然后中途没人改动时改掉。其他的stdatomic.h

int table = YES;

void lock() {
retry:
    int got = xchg(&table, NOPE);
    if (got == NOPE) goto retry;
    assert(got == yes);
}
void unlock() {
    xchg(&table, YES);
}

int locked = 0;
void lock() {
    while( xchg(&locked, 1));
}
void unlock() { xchg(&locked, 0); }

x86的lock指令的实现。 L1 cache的MESI状态协调。

RISC-V的另一种原子操作的设计。

本质是load exec store的原子性。

Load-Reserved/Store-Conditional (LR/SC)

使用场景： 短临界区，少竞争下，防止线程被切换（程序自身做不到，OS控制持锁时OS禁止中断和抢占）。 操作系统内核的并发数据结构使用spin lock。

缺陷： 空转。 获得自旋锁的线程被os切换出去了（OS也很难保证，虚拟机不能关中断）。

互斥锁（mutex lock）。 🔗

天黑请闭眼，停止操作。

长临界区的互斥。写文件。把锁的实现放到操作系统里。通过系统调用访问locked。

  • 自旋锁
    • fast path。 xchg 成功，进入
    • slow path。 xchg 失败失败，自旋等待
  • 互斥锁，通过系统调用访问locked
    • slow path。更快。上锁失败的线程不再占用cpu
    • fast path。上锁成功也需要syscall到内核态

futex（Fast Userspace muTexes） 🔗

借鉴 自旋锁和互斥锁的优点。 设计futex的人第一次也没有写对。

  • fast path。 一条原子指令，上锁成功后立即返回。
  • slow path。 上锁失败，执行系统调用睡眠。

互斥是竞争。 同步是协作。

同步问题： 生产者-消费者； 哲学家吃饭。 同步的实现方法： 条件变量, 信号量。

生产者-消费者问题。

 void Tproduce() { while(1) printf("(");}

同步： 1 等到有空位时才能打印左括号。2 等到能配对时才能打印右括号。

#include "thread.h"
#include "thread-sync.h"

int n, count = 0;
mutex_t lk = MUTEX_INIT();

void Tproduce() {
    while(1){
    retry:
        mutex_lock(&lk);
        if (count == n)  {
            mutex_unlock(&lk);
            goto retry;
        }
        count++;
        printf("(");
        mutex_unlock(&lk);   
    }
}

void Tconsumer() {
    while(1){
    retry:
        mutex_lock(&lk);
        if (count == 0){
            mutex_unlock(&lk);
            goto retry;
        }
        count--;
        printf(")");
        mutex_unlock(&lk);
    }
}

不管缓冲区空还是满，每个线程都会拿来加锁。

条件变量：万能同步的方法。

  • 条件变量API：
    • wait(cv, mutex) 调用时必须已经获得mutex，则释放mutex，进入睡眠;
    • signal(cv) 唤醒等待cv的线程。
    • broadcast/notifyAll(cv) 唤醒All。
void Tproduce() {
    while(1) {
        mutex_lock(&lk);
        if ( !count !=n ) {
            mutex_unlock_and_sleep(&lk);
            // mutex_lock(&lk);
        }
        count++;
        printf("(");
        wakeup();
        mutex_unlock(&lk);
    }
}
void Tconsumer() {
    while(1) {
        mutex_lock(&lk);
        if (! count > 0) {
            mutex_unlock_and_sleep(&lk);
        }
        count--;
        printf(")");
        wakeup();
        mutex_unlock(&lk);
    }
}

int n, count = 0;
mutex_t lk = MUTEX_INIT();
cond_t cv = COND_INIT();
void Tproduce() {
    while(1) {
        mutex_lock(&lk);
        if (count == n) { // 改while

        mutex_lock(&lk);
        if (count == 0) { // 改while

        }
        printf(")"); count--;
        cond_signal(&cv);
        mutex_unlock(&lk);
    }
}
