etcd arch 🔗


etcd中的事务隔离级别的实现。 在server的mvcc基础上， 在client上事务。 client v3中concurrency包。 mutex.go 分布式锁的实现 stm.go 在客户端实现的事务框架

    stm（software transactional memory）
type STM interface {
    Get(key)  // 返回value，并insert到txn的read set中。
    Put(key, value) //写kv，并insert到txn的write set中。
    ......

}

struct stm 实现了repeat read隔离级别，STM接口。 rset readset， wset writeset， conflicts func() []v3.Cmp 冲突检测。 读请求： 先以本地为准，没有则去server获取，并缓存， 以此来实现可重复读。 可重复读的实现关键点： 1. readset缓存已经读过的数据，保持可重复读。 2. readset中数据的modrevision与server中的modrevision 相同，保持事务读到的数据是server中最新的。 读已提交（read committed）实现上 仅仅是conflicts没有做 检测。

MVCC 的实现： 内存treeIndex（B树）， boltdb（B+树）
